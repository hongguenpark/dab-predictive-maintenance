# 기능 DAB 배포 가능 여부 DAB 리소스 타입 샘플 코드 (GitHub) 배포된 리소스 (Databricks) 주요 사항
1 Unity Catalog Function 간접 지원 schemas + jobs 📁 GitHub 🔧 Job UC Function 자체는 DAB 리소스 타입이 아님. Schema는 DAB로 관리 가능하나, 함수 생성은 SQL 노트북을 Job으로 실행하여 배포. 3개 함수 구현: 진동 상태 판별, RUL 추정, 정비 우선순위 점수
2 Notebook ✅ 지원 jobs (notebook_task) 📁 GitHub 📓 EDA Job 노트북은 DAB의 sync를 통해 워크스페이스에 자동 업로드됨. Job의 notebook_task로 실행 가능. EDA 노트북: 센서 데이터 생성, 통계 분석, 이상 탐지
3 Job (Workflow) ✅ 지원 jobs 📁 GitHub ⚙️ 일일 분석 Job DAB의 핵심 리소스. 멀티태스크 Job 지원 (의존성 포함). 스케줄(cron), 알림, 클러스터 설정 모두 YAML로 관리. 3단계 파이프라인: 데이터 수집 → 분석 → 알림 생성
4 DLT Pipeline
(Lakeflow)		 ✅ 지원 pipelines 📁 GitHub 🔄 DLT Pipeline Medallion 아키텍처 완벽 구현. pipelines 리소스로 직접 선언. Unity Catalog 통합 지원. DLT Expectations로 데이터 품질 검증 포함. Bronze(원시) → Silver(정제) → Gold(집계) 3단계
5 Lakebase ❌ 미지원 해당 없음 📁 README - DAB 리소스 타입 없음. Lakebase(관리형 PostgreSQL)는 REST API 또는 SDK로만 프로비저닝 가능. 대안: App 내에서 Lakebase SDK 사용, 또는 노트북에서 API 호출
6 Databricks App ✅ 지원 apps 📁 GitHub 워크스페이스 제한(300개) apps 리소스로 직접 배포 가능. source_code_path로 소스 동기화, resources로 SQL Warehouse/Secret 바인딩. ⚠️ 이 워크스페이스는 App 300개 제한에 도달하여 실제 배포 불가. FastAPI 대시보드 구현 (장비 상태 모니터링)
7 AI/BI Dashboard ✅ 지원 dashboards 📁 GitHub 📊 Dashboard dashboards 리소스로 직접 배포. Lakeview JSON 형식으로 대시보드 정의. databricks bundle generate dashboard로 기존 대시보드 export 가능. 장비 상태 테이블, 상태 분포 차트, 진동 추이 라인차트 포함
8 Genie ❌ 미지원 해당 없음 📁 README - DAB 리소스 타입 없음. Genie Space는 REST API(/api/2.0/genie/spaces)로만 생성 가능. 대안: AI/BI Dashboard에 Genie 기능이 통합되어 있으므로 Dashboard 배포로 대체 가능
9 Metric View 간접 지원 jobs (SQL notebook) 📁 GitHub 📐 Metric View Job Metric View 자체는 DAB 리소스 타입이 아님. SQL 노트북으로 CREATE VIEW 문을 실행하여 배포. OEE(종합설비효율), MTBF(평균고장간격), 가동률 메트릭 구현
10 ML Custom Model ✅ 지원 experiments + registered_models 📁 GitHub 🧪 Experiment 🤖 Model 📈 Training Job Experiment + Registered Model + 학습 Job을 모두 DAB로 관리. MLflow 실험 추적과 UC 모델 레지스트리 연동. LightGBM 기반 RUL(잔여수명) 예측 모델. Feature: 진동, 온도, 압력, 전류, 운전시간
11 Model Serving ✅ 지원 model_serving_endpoints 📁 GitHub 모델 학습 후 배포 가능 model_serving_endpoints로 직접 배포 가능. served_entities로 모델 버전 지정, scale_to_zero 설정 가능. AI Gateway inference table로 추론 로그 자동 수집. ⚠️ 모델이 먼저 등록되어야 endpoint 생성 가능
12 AgentBricks
Knowledge Assistant		 ❌ 미지원 해당 없음 📁 GitHub 📚 KB 설정 Job AgentBricks Knowledge Assistant는 DAB 리소스 타입 없음. UI에서만 생성 가능 (AI/BI > AgentBricks). 대안: DAB Job으로 지식 베이스 데이터 + Vector Search 인덱스 준비 후 UI에서 연결. 8개 정비 매뉴얼 문서 준비 완료
13 AgentBricks
Supervisor Agent		 ❌ 미지원 해당 없음 📁 GitHub 🤖 Agent 설정 Job AgentBricks Supervisor Agent는 DAB 리소스 타입 없음. UI에서만 생성 가능. 대안: Mosaic AI Agent Framework + MLflow로 코드 기반 에이전트 개발 후 model_serving_endpoints로 배포. 3개 Sub-Agent 설계: 지식검색, 데이터분석, RUL예측

미지원 (3개) Lakebase, Genie, AgentBricks (Knowledge Assistant, Supervisor Agent) 해당 없음 REST API 또는 UI에서만 관리 가능. DAB Job에서 API 호출로 자동화 가능
🏭 예측 정비 (Predictive Maintenance) 시나리오 개요

시나리오 설명

공장의 8대 핵심 설비(CNC 선반, 프레스기, 컨베이어, 로봇암, 펌프, 압축기, 냉각기, 보일러)에서 IoT 센서(진동, 온도, 압력, 전류)로 실시간 데이터를 수집하여 설비 상태를 모니터링하고, ML 모델로 잔여 수명(RUL)을 예측하여 사전 정비를 수행하는 시나리오입니다.

데이터 흐름

IoT 센서 → DLT Pipeline (Bronze→Silver→Gold) → 일일 분석 Job → ML 모델 학습/추론 → AI/BI Dashboard + 알림
정비 매뉴얼 → Vector Search → Knowledge Assistant → Supervisor Agent

주요 UC Functions

  • 🔹 assess_vibration_status() - 진동/온도 기반 장비 상태 판별 (정상/주의/위험/긴급)
  • 🔹 estimate_rul() - 잔여 유효 수명 추정 (시간 단위)
  • 🔹 maintenance_priority_score() - 정비 우선순위 점수 (0~100)
